# Обзор API aaPanel

[English version](../en/overview.md) · [⌂ Главная](../../README.ru.md)

aaPanel (международная версия панели BT/宝塔) — это веб-панель управления Linux-сервером: сайты, базы данных, FTP, SSL, фаервол, бэкапы и т.д. Почти всё это можно автоматизировать через HTTP API, не заходя в веб-интерфейс.

Этот репозиторий документирует **управление Node.js-проектами** через API — раздел, который в официальной документации толком не описан.

---

## Две схемы доступа к API

У aaPanel фактически **два разных способа** обращаться к серверу, и их важно не путать.

### 1. Официальный API (постоянный ключ `api_sk`)

- Включается в панели: **Настройки → API**, там же генерируется ключ `api_sk` и настраивается白名单 (whitelist) разрешённых IP.
- Аутентификация по подписи на каждый запрос:
  `request_token = md5( request_time + md5(api_sk) )`
- **Ключ постоянный** — не меняется, подходит для автоматизации.
- ⚠️ **Покрывает не все функции.** Официальная документация ([api-list](https://www.aapanel.com/docs/api/api-list.html), [PDF](https://www.aapanel.com/Document/api.pdf)) описывает в основном «классику»: систему, сайты, БД, FTP, SSL, cron, фаервол, DNS. Управление Node.js-проектами там не описано.

### 2. Внутренний интерфейс панели (сессионный токен)

- Это тот же API, которым пользуется веб-интерфейс панели в браузере.
- В URL фигурирует **сессионный токен** `apsess_...` — временный «пропуск», который панель выдаёт браузеру после входа.
- Через него доступно **всё**, что умеет панель, включая `/v2/project/nodejs/...`.
- ⚠️ Токен **временный**: меняется при перелогине и со временем истекает. Подробнее — [authentication.md](authentication.md).

> 💡 **Официальная позиция aaPanel:** всё, что не описано в документации, предлагается находить самостоятельно — открыть в браузере DevTools (вкладка Network), выполнить нужное действие в панели и посмотреть, какой запрос она отправляет. Методы в этом репозитории получены именно так.

## Дисклеймер

Это **неофициальная**, community-документация. Раздел Node.js-проектов основан на наблюдении за реальными запросами панели и может измениться при обновлении aaPanel. Проверяйте поведение на своей версии панели. Официальная документация — на [aapanel.com/docs](https://www.aapanel.com/docs/).
