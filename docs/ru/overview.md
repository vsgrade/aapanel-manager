# Обзор API aaPanel

[English version](../en/overview.md) · [⌂ Главная](../../README.ru.md)

aaPanel (международная версия панели BT Panel) — это веб-панель управления Linux-сервером: сайты, базы данных, FTP, SSL, фаервол, бэкапы, Node.js-проекты и т.д. Почти всё это можно автоматизировать через HTTP API.

Этот репозиторий документирует **управление Node.js-проектами** и **мониторинг сервера** — с упором на то, что плохо описано официально, но проверено на живой панели (v8).

---

## Две схемы доступа

### 1. Ключ `api_sk` (рекомендуется для приложений)

- Включается: **Настройки → API** → сгенерировать ключ → добавить IP в whitelist.
- Подпись: `request_token = md5( request_time + md5(api_sk) )`.
- **Ключ постоянный** — не протухает.
- ✅ **Проверено:** работает на корне (без защитного входа) и покрывает **и** официальные эндпоинты (`/system?action=…`), **и** внутренние (`/v2/project/nodejs/…`). То есть одним ключом — и сервер, и Node.js-проекты.

### 2. Сессия (для разведки в браузере)

- Тот же доступ, что у залогиненного браузера: токен `apsess_...` в URL + `x-http-token` + cookie.
- ⏳ Временная (истекает) — для приложения не годится, но удобна, чтобы **найти** нужный запрос.

Подробности обоих способов — [authentication.md](authentication.md).

---

## 🔍 Рецепт «разведка → исполнение»

Официальный совет aaPanel для недокументированных функций: открой панель в браузере → DevTools (Network) → нажми кнопку → посмотри запрос → повтори его в коде. Мы это проверили: **тот же путь и тело** работают через постоянный ключ `api_sk` (меняется только авторизация). См. [authentication.md](authentication.md).

## Дисклеймер

Неофициальная community-документация. Часть эндпоинтов получена наблюдением за запросами панели и проверена на конкретной версии (aaPanel v8). Поведение может меняться между версиями — сверяйтесь со своей панелью. Официальная дока — [aapanel.com/docs](https://www.aapanel.com/docs/).
