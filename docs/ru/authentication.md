# Аутентификация

[English version](../en/authentication.md) · [⌂ Главная](../../README.ru.md)

У aaPanel есть **два рабочих способа** обращаться к API. Оба проверены на живой панели (v8).

## Какой выбрать

| | 🔑 Ключ `api_sk` | 🍪 Сессия |
|--|------------------|-----------|
| Для чего | **приложения, автоматизация** | разведка в браузере, ручные пробы |
| Срок жизни | ♾️ постоянный | ⏳ временный (протухает) |
| URL | корень, без защитного входа | с `/apsess_.../` в пути |
| Покрытие | система + Node.js + сайты + БД… | всё, что умеет панель |

> 💡 **Проверено:** ключ `api_sk` дотягивается **и** до официальных (`/system?action=…`), **и** до внутренних (`/v2/project/nodejs/…`) эндпоинтов. То есть одним постоянным ключом можно управлять всем — для приложения это правильный выбор.

## Способ 1 — ключ `api_sk` (рекомендуется)

### Настройка в панели
1. **Настройки → API** → включить интерфейс.
2. Сгенерировать `api_sk`.
3. Добавить **IP машины**, с которой пойдут запросы, в whitelist.

### URL
```
https://<СЕРВЕР>:<ПОРТ>/<эндпоинт>
```
⚠️ **На корне** — без защитного входа (`/xxxxxxxx`) и без `apsess`-токена.

### Подпись запроса
В тело POST добавляются два поля:

| Поле | Значение |
|------|----------|
| `request_time` | текущий Unix-таймстамп |
| `request_token` | `md5( request_time + md5(api_sk) )` |

Псевдокод:
```
request_time  = 1780677549
request_token = md5( "1780677549" + md5(api_sk) )
```

### Пример (curl + bash)
```bash
BASE="https://<СЕРВЕР>:<ПОРТ>"
SK="<API_SK>"
T=$(date +%s)
SK_MD5=$(printf '%s' "$SK" | md5sum | cut -d' ' -f1)
TOKEN=$(printf '%s' "$T$SK_MD5" | md5sum | cut -d' ' -f1)

# Node.js: список проектов
curl -k -X POST "$BASE/v2/project/nodejs/get_project_list" \
  --data-urlencode "request_time=$T" \
  --data-urlencode "request_token=$TOKEN" \
  --data-urlencode 'data={"p":1,"limit":10}'

# Система: ресурсы сервера
curl -k -X POST "$BASE/system?action=GetSystemTotal" \
  --data-urlencode "request_time=$T" \
  --data-urlencode "request_token=$TOKEN"
```

Готовая обёртка на TypeScript: [`examples/javascript/aapanel-client.ts`](../../examples/javascript/aapanel-client.ts).

## Способ 2 — сессия (для разведки в браузере)

Тот же доступ, что у залогиненного браузера. Удобно, чтобы **найти** нужный запрос, но для приложения не годится — токен временный.

### URL
```
https://<СЕРВЕР>:<ПОРТ>/<SESSION_TOKEN>/<эндпоинт>
```
`<SESSION_TOKEN>` — кусок `apsess_...` из адресной строки браузера после входа.

### Заголовки
| Заголовок | Откуда |
|-----------|--------|
| `x-http-token` | из любого запроса панели (DevTools → Network → Headers) |
| `Cookie` | session-cookie браузера |

> ⚠️ И токен, и cookie **временные**: меняются при перелогине и со временем истекают. Поэтому в коде их не зашивают — для постоянной работы используйте `api_sk` (Способ 1).

## 🔍 Рецепт «разведка → исполнение» (официальный способ aaPanel)

Многие функции панели официально не описаны. aaPanel предлагает находить их самостоятельно — и это работает:

1. **Разведка (браузер):** открой панель → `F12` → вкладка **Network** → нажми нужную кнопку → посмотри ушедший запрос (путь + тело).
2. **Исполнение (код):** возьми **тот же путь и тело**, но на корне и с авторизацией ключом (`request_time` + `request_token`) вместо cookie.

| | В браузере (сессия) | Через ключ (API) |
|--|--------------------|------------------|
| Путь и тело | **одинаковые** | **одинаковые** |
| Авторизация | cookie + `x-http-token` | `request_time` + `request_token` |
| Защитный вход в URL | есть | **не нужен** |

## Формат тела запроса
`Content-Type: application/x-www-form-urlencoded`.
- **Node.js**-эндпоинты: параметры в поле `data=<URL-encoded JSON>`.
- **Системные**: действие в query — `?action=GetSystemTotal`.
- Для ключа к любому запросу добавляются `request_time` и `request_token`.

## SSL: самоподписанный сертификат
Панель обычно с самоподписанным сертификатом: в `curl` — флаг `-k`, в Node.js — отключить проверку для этого хоста. В production лучше добавить доверенный CA панели, а не отключать проверку глобально.

## Безопасность
- `api_sk`, токены, cookie — **секреты**. Только `.env` (см. [`.env.example`](../../.env.example)), никогда в git.
- Ключ `api_sk` даёт **полный доступ к серверу** → держать только на бэкенде, плюс whitelist по IP.
- Все запросы — со стороны сервера, не из браузера пользователя (иначе CORS, утечка секретов, whitelist по IP).
